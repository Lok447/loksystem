AI Native Collaboration Platform

把本地工作区、智能体协作与企业交付，收束到同一个 LokSystem 工作台。

LokSystem 面向真实生产环境打造，不只是聊天入口，也不只是单一命令行代理。 它将本地 Agent 工作台、多 Agent 协同、远程 WebUI、企业渠道接入、Office 原生产出与 MCP 扩展整合为统一系统， 让个人与团队都能围绕同一工作空间持续推进任务、交付结果。

当前版本能力概览

  • 本地工作台：围绕文件、任务、预览与对话展开，Agent 在你的真实目录中工作。
  • 多 Agent 协同：支持 Leader / Teammate 分工、任务拆解、并行执行与结果回传。
  • 远程触达：支持 WebUI 与微信、企微、飞书、钉钉等渠道接入，便于移动端和异地访问。
  • Office 交付：支持 PPT、Word、Excel、PDF 等原生产出与预览闭环。
  • 可扩展系统：统一管理 MCP、技能与工具链，降低多 Agent 配置成本。

整体介绍

LokSystem 的定位是“AI 原生协同平台”，不是把更多模型堆在同一个对话框里，而是把任务、上下文、工具、产物与协作链路真正打通。

面向真实工作流

从代码修改、资料整理、日报周报，到演示文稿、文档、表格与 PDF 交付，LokSystem 更强调完整链路，而非单轮回答。

产品对比

如果把“AI 聊天工具”“单一 CLI Agent”和“LokSystem 协同平台”放在一起看，差异主要体现在工作空间、协作机制与交付闭环。

对比项 普通 AI 聊天工具 单一 CLI Agent LokSystem
工作空间 以对话为主，上下文依赖手动粘贴 面向单终端会话 围绕本地 Workspace 持续工作，文件、任务、预览同屏联动
协作方式 通常为单模型单线程 通常为单 Agent 执行 支持多 Agent 协同、任务拆解、并行推进与结果回收
远程使用 多为网页聊天入口 依赖额外远控方案 支持 WebUI + 微信 / 企微 / 飞书 / 钉钉等渠道触达
交付物类型 偏文本回答 偏代码与终端输出 兼顾代码、文档、表格、PPT、PDF 等生产级结果
工具扩展 能力边界固定 各 Agent 常需单独配置 统一管理 MCP、技能与工具链，复用配置更高效
持续执行 多为即时问答 脚本化后可执行，但协作弱 适合长任务、定时任务与持续推进型工作流

快速开始

如果你第一次接触 LokSystem，可以先从四步跑通完整体验：打开工作台、选择 Agent、接入工具、产出结果。

01 安装并启动

启动 LokSystem 后，先熟悉系统设置、工作区与关于页入口，确认本地工作目录与基础环境。

02 选择协作方式

可先使用内置 Lok CLI，也可根据需求接入 Codex、OpenCode、Qwen Code、OpenClaw 等已有 Agent 工作流。

03 配置能力扩展

在设置中按需接入 MCP、技能、WebUI 或远程渠道，让 Agent 从对话能力扩展到真实执行能力。

04 开始交付任务

从代码处理、资料整理、表格分析、PPT 生成等实际任务开始，逐步建立你的个人或团队工作流。

常见问题

以下问题来自 LokSystem 常见上手场景，帮助你快速判断它是否适合当前团队与业务流程。

LokSystem 和普通 AI 客户端最大的区别是什么？

LokSystem 更强调“协同工作台”而不是“对话框”。它把文件、任务、预览、远程访问、多 Agent 协作和交付物产出整合进同一个系统，适合持续推进真实任务。

一定要先安装很多 CLI 工具才能使用吗？

不一定。LokSystem 可以从内置能力开始使用；如果你已经有 Codex、OpenCode、Qwen Code、OpenClaw 等环境，再接入后能获得更完整的多 Agent 工作流。

它更适合个人还是团队？

两者都适合。个人可以把它当作本地 AI 工作台，团队则更适合利用多 Agent、WebUI、企业渠道和 Office 交付能力，把工作流延展到多人协同与远程触达。

LokSystem 适合哪些典型任务？

典型包括代码与文档改造、知识整理、汇报材料生产、表格分析、PPT 输出、长期任务托管，以及需要跨工具持续推进的复杂工作。
