# Windows 安装 / 升级 / 回滚指南

适用对象：Windows x64 用户。

## 1. 安装前检查

1. 确认下载文件名符合版本规范，例如：`LokSystem-2.0.0-win-x64.exe`。
2. 如从源码本地打包，产物默认位于 `out/`。
3. 如设备上已有旧版本，请先确认当前 workspace、配置与数据库目录已备份。

## 2. 全新安装

1. 从 GitHub Releases 下载 `.exe` 安装包。
2. 双击安装程序，按向导完成安装。
3. 首次启动后检查：图标、About 页、启动页、默认语言、静态官网页是否正常。

## 3. 升级步骤

1. 关闭正在运行的 LokSystem。
2. 备份用户数据目录、数据库文件、自定义助手配置和 workspace 映射。
3. 运行新的 `.exe` 安装包覆盖安装。
4. 升级后至少验证：应用可启动、历史数据可见、旧助手未丢失、旧 workspace 配置仍可打开、WebUI 与桌面端同步正常。

## 4. 回滚步骤

1. 关闭当前版本 LokSystem。
2. 保留最新备份的数据目录与配置目录。
3. 卸载当前版本（如安装器要求）。
4. 安装上一个稳定版本。
5. 用备份数据恢复数据库、配置和 workspace 映射。
6. 回滚后重点检查：应用能否启动、历史会话是否还在、旧助手与旧 workspace 是否可用、主链路是否恢复正常。

## 5. 升级失败时优先排查

- 安装包文件名不正确或版本号不一致
- 桌面快捷方式仍指向旧路径
- 本地脚本、日志目录、缓存目录仍引用旧盘符
- 数据迁移未完成，导致数据库或配置读取失败

## 6. 2.0.0 / v2.1.3 升级后专项验收

1. 登录与认证
   - 桌面端首次打开需要登录
   - WebUI 与桌面端均可使用默认管理员账号 `admin / Admin@123` 登录
2. 历史与配置
   - 已删除的内置 assistant 不会在重启后恢复
   - 旧 provider / assistant 配置不会被重复迁移覆盖
3. 对话链路
   - 新建会话、发送消息、切换会话正常
   - 切换会话时未完成上传会自动取消，不会串到其他会话
4. 工作区
   - paste / drop / 手动附件后仍保留原 workspace
   - workspace 文件树可刷新、可 reveal、可外部打开
5. 故障处理
   - 若 backend/core 启动失败，桌面端能看到明确错误信息，而不是直接退出
   - 退出应用后不残留明显子进程
