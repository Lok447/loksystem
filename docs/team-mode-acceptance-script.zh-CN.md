# Team Mode 真实用户验收脚本

目标：覆盖“leader 派任务给 2 个成员 → 一个完成、一个失败 → leader 汇总结果”的真实回归路径。

## 前置准备

1. 启动桌面端并确认 Team Mode 可用。
2. 准备 1 个 leader、2 个 teammate。
3. 为团队设置同一个 workspace，方便观察共享文件变化。
4. 打开团队会话和 workspace 面板，保证可以同时看到成员状态和文件变化。

## 验收步骤

1. 让 leader 创建一个团队任务，例如“整理本周发布说明并生成摘要”。
2. leader 把任务拆成两个子任务：
   - 成员 1：整理版本变更，写入 `release-notes.md`
   - 成员 2：汇总已知风险，写入 `risk-summary.md`
3. 确认两个成员都收到任务，状态进入 `active`。
4. 等待成员 1 完成子任务，确认 leader 收到完成通知，workspace 出现 `release-notes.md`。
5. 人为制造成员 2 失败，例如关闭对应 CLI 进程或提供必然失败的指令。
6. 确认成员 2 状态变为 `failed`，leader 收到失败原因和任务重分配提示。
7. 确认未完成任务不再永久挂在失败成员名下。
8. 让 leader 汇总：已完成内容、失败内容、补救动作。

## 通过标准

- leader 能成功派发给 2 个成员
- 至少 1 个成员完成后，workspace 能看到产物
- 至少 1 个成员失败后，leader 能收到失败提示
- 未完成任务不会永久挂在失败成员名下
- leader 最终能产出结构化总结，不会因为成员失败而卡死
